How NetRadar Was Started

This was one of the more interesting projects I have done in a very long while. I loved it. The idea came to me in the midst of the war between Iran and U.S.-Israel in April 2026. During this war, I was living in Iran and access to the international internet was cut off. Eventually after a very fragile cease fire, the Iranian government eased off the restrictions. I gained limited access to the Internet. But the trouble was I didn't know which services were working and which were still banned. That was where the Idea of NetRadar came to me.

A Service to the General Public

The initial motivation for creating this service was three-fold. First and foremost, it gave me a chance to build a web app from ground up, from end to end, without help of a single person. I had a previous experience in building a service called BluLexi - I'm going to talk about it in the future. There, I relied heavily on people to develop the project; I was only the master mind, the center of ideation. So, NetRadar gave me a solid chance of proving my coding + prompt engineering worth to myself.

Secondly, I had a prior knowledge of Python and building Flask apps, but I was illiterate in frontend development. Of course I knew the frameworks, the languages, the overall knowledge of what was going on, but how to build a working frontend app was missing from my repertoire of programming skills. This was a good occasion to see how the frontend portion of a website is created and managed. I chose Next.js and started playing with it. It was challenging. I can even say it was overwhelming, because the first time I initialized it in a directory, there were too many files and folders, too many variables to configure. But I finally got the hang of it, and after 9 failed efforts, I got what I was looking for. Since I didn't know how it all worked, I relied heavily on AI to do the job for me, until I was skillful enough to take over and make changes. I'm still a newbie after finishing this project, but I consider myself to be more literate in frontend development.

Finally, I thought of this project as a service to the general Iranian public, since it solved a terrible pain. The problem with government restrictions at the time was that there was no way of getting to know which international services were unbanned and could be used. For example, I got to know GitHub was accessible by sheer chance as I reluctantly clicked on its bookmark on my browser to see what happens. And there it was, my beautiful github homepage. So, a web service that could check if a set of services were up and running was invaluable for people and the health of their psyche. So, I embarked on this journey for the good of others, and satisfying my personal ambitions; a classic protagonist journey :)

A Storm of Ideas

If you're an engineer like me, you've had episodes in your life that you wanted to over-engineer a solution. This project was one of those episodes. At first, the goal was to ping some services and show the user ping statistics. Then I realized some services might be up and running, but you might not be able to ping them, so I used HTTP requests to see if they were actually up or not. Soon, the database turned into a table with several fields that the program checked against a list of Iranian and internation services. The project grew into a complex web service soon, in a matter of days. Then I thought, what if I shared it with others? What if others could use this service for their personal needs? What if I released it as an open-source web app?

I did that. I decided to release it as an open-source web app - a service availability website called NetRadar. But another version of this app, an rtl-Persian-tuned version for the Iranian audience was kept in a private sister repo to serve as a website later on at netvasle.ir. This was when I could finally find a good VPN that could get me a narrow and unstable stream of international internet access. I shared the news with my uncle, Rod, who was equally thrilled about the project.

At that moment I did not lack for development ideas. But my uncle struck me with 2 great ones: adding a TUI and a CLI. I was even more excited to implement them than finishing off what I'd started. You know, I had a GUI, a graphical user interface, the web app. But adding a Textual UI, a TUI, was even cooler. So I immediately started doing that. The most important use case for a TUI is for when the project is deployed on a remote server and you just want to check if it all's running well. And it's cool too! The CLI was more meant to be used for the machines, the AI agents, interacting with this project on a local system. I even wrote machine friendly documentation for it. And released all of it as v0.1.0. This project took exactly 14 days from ideation to v0.1.0.

Although some ideas were still there to improve it and add new features for future releases, but at the moment of writing this blog post, I feel like I'm largely done with this project and I have done my share of good to the general public and publishing open-source software! Just kidding. Doing this project felt good. And I liked how it turned out.

The Future?

Well, there are several other projects I have open and I'm working on. This was just one of many ideas that ended up in a a good place, and not in the trash bin. It's not over yet. I'll add even more to it over time. I'm serious at creating something that is truly safe and serves everyone, and everyone can contribute to make it even better. I have not even done the research to see if there are similar open-source software available or not. I have satisfied the 3 main reasons to do this project, and that's the most important thing for me.

The Iranian version is not published yet, unfortunately. But I'm working on it. Maybe it's up and running when you read this post. I'll talk about it in the future. The future of this project seems to be bright. Let's be a judge of this over time.
